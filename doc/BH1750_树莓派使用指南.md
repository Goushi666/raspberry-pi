# BH1750 光照强度传感器 - 树莓派使用指南

## 1. 传感器概述

BH1750FVI 是一款数字型光照强度传感器，通过 I2C 接口与主控通信。

| 参数 | 值 |
|------|-----|
| 芯片型号 | BH1750FVI |
| 通信协议 | I2C |
| I2C 地址 | 0x23（ADDR 引脚接地）/ 0x5C（ADDR 引脚接 VCC） |
| 测量范围 | 1 ~ 65535 lx |
| 工作电压 | 3.3V / 5V |
| 分辨率 | 1 lx |

## 2. 硬件接线

### 2.1 模块接口说明

该光照强度传感器模块有 **两个接口 J1 和 J2**，在原始 UP 派平台上分别连接扩展板的 P1 和 P8 端口。两个接口功能不同，**缺一不可**。

| 接口 | 引脚 | 功能说明 | 原平台对应 |
|------|------|----------|------------|
| **J1** | VCC (5V) | 电源正极 | UP 派扩展板 P1 |
| | SCL | I2C 时钟线 | |
| | SDA | I2C 数据线 | |
| | GND | 电源地 | |
| **J2** | VCC (5V) | 电源正极 | UP 派扩展板 P8 |
| | DVI (GPIO) | 传感器使能引脚（SDA/SCL 参考电压 + 内部寄存器异步重置） | |
| | GND | 电源地 | |

> **J1** 负责 I2C 通信和供电，**J2** 负责 DVI 使能引脚控制。DVI 引脚必须拉高，传感器才能正常工作。

### 2.2 J1 和 J2 是否需要共地、共同供电？

**是的，必须共地。** J1 和 J2 的 GND 都要接到树莓派的 GND，确保信号参考电平一致。供电方面，两个接口的 VCC 可以接到树莓派的同一个电源引脚（或分别接不同的 5V/3.3V 引脚，树莓派有多个电源引脚可用）。

简单理解：
- **共地是必须的** —— 不共地 I2C 通信会异常，DVI 使能也无法正常工作
- **供电可以共用** —— 从树莓派的同一个 5V 或 3.3V 引脚取电即可，BH1750 功耗很低（<1mA）

### 2.3 与树莓派接线

树莓派使用 I2C-1 总线（GPIO2/GPIO3），完整接线方式如下：

```
J1 接口（I2C 通信）                    树莓派 GPIO 排针
┌──────────┐                         ┌─────────────────────┐
│  VCC     │────────────────────────>│  Pin 2  (5V)        │
│  SCL     │────────────────────────>│  Pin 5  (GPIO3/SCL) │
│  SDA     │────────────────────────>│  Pin 3  (GPIO2/SDA) │
│  GND     │──────────┐             │                     │
└──────────┘          ├────────────>│  Pin 6  (GND)       │
                      │             │                     │
J2 接口（DVI 使能）    │             │                     │
┌──────────┐          │             │                     │
│  VCC     │────────────────────────>│  Pin 4  (5V)        │ ← 或与J1共用Pin2
│  DVI     │────────────────────────>│  Pin 1  (3.3V)      │ ← 直接接高电平使能
│  GND     │──────────┘             │                     │ ← 与J1共地
└──────────┘                         └─────────────────────┘
```

**接线汇总表：**

| 模块接口 | 模块引脚 | 树莓派引脚 | 说明 |
|----------|----------|------------|------|
| J1 | VCC | Pin 2 (5V) | 供电 |
| J1 | SCL | Pin 5 (GPIO3/SCL1) | I2C 时钟 |
| J1 | SDA | Pin 3 (GPIO2/SDA1) | I2C 数据 |
| J1 | GND | Pin 6 (GND) | 共地 |
| J2 | VCC | Pin 4 (5V) | 供电（可与 J1 VCC 共用） |
| J2 | DVI | Pin 1 (3.3V) | 直接拉高使能传感器 |
| J2 | GND | Pin 6 (GND) | 共地（与 J1 GND 接同一个 GND） |

### 2.4 接线实物参考图

```
树莓派 40Pin 排针（局部）
┌─────┬─────┐
│Pin 1│Pin 2│   Pin 1 = 3.3V  ← J2 DVI（拉高使能）
│ 3V3 │ 5V  │   Pin 2 = 5V    ← J1 VCC
├─────┼─────┤
│Pin 3│Pin 4│   Pin 3 = SDA   ← J1 SDA
│ SDA │ 5V  │   Pin 4 = 5V    ← J2 VCC（或与Pin2共用）
├─────┼─────┤
│Pin 5│Pin 6│   Pin 5 = SCL   ← J1 SCL
│ SCL │ GND │   Pin 6 = GND   ← J1 GND + J2 GND（共地）
├─────┼─────┤
│Pin 7│Pin 8│
│ ... │ ... │
└─────┴─────┘
```

> **注意事项：**
> - **J1 和 J2 的 GND 必须共地**，都接到树莓派的 GND 引脚。
> - J2 的 DVI 引脚是传感器使能信号，**必须接高电平**（3.3V），否则传感器无法工作。在原平台上这由内核驱动通过 GPIO 控制拉高，在树莓派上直接接 3.3V 即可。
> - 树莓派 GPIO 为 3.3V 电平，BH1750 兼容 3.3V，I2C 信号线直接连接即可，**不需要电平转换**。
> - 模块原设计使用 5V 供电（J1/J2 的 VCC 标注为 5V），树莓派的 5V 引脚（Pin 2/4）可直接供电。若使用 3.3V 供电也可以工作。
> - ADDR 引脚（若模块上有，一般已通过电路板默认接 GND）对应 I2C 地址 `0x23`。

```bash
dtparam=i2c_arm=on

# 确保 /etc/modules 中加载了 i2c 模块：
echo "i2c-dev" | sudo tee -a /etc/modules
```

重启生效：

```bash
sudo reboot
```

## 4. 代码实现

### 4.1 方案一：C 语言直接操作 I2C 设备（推荐）

此方案基于原始源码改写，直接在用户态通过 `/dev/i2c-1` 设备节点与 BH1750 通信，**无需编写内核驱动**，适合树莓派。

#### 4.1.1 完整代码 `bh1750_read.c`

```c
#include <fcntl.h>
#include <stdio.h>
#include <string.h>
#include <unistd.h>
#include <sys/ioctl.h>
#include <linux/i2c-dev.h>

/* I2C 设备节点 */
#define I2C_DEV       "/dev/i2c-1"
/* BH1750 I2C 地址（ADDR 引脚接 GND） */
#define BH1750_ADDR   0x23

/* BH1750 指令定义 */
#define BH1750_POWER_ON    0x01   /* 等待测量指令（上电） */
#define BH1750_RESET       0x07   /* 重置数据寄存器 */
#define BH1750_CONT_H_RES  0x10   /* 连续高分辨率模式，1lx，测量时间120ms */
#define BH1750_CONT_H_RES2 0x11   /* 连续高分辨率模式2，0.5lx，测量时间120ms */
#define BH1750_CONT_L_RES  0x13   /* 连续低分辨率模式，4lx，测量时间16ms */
#define BH1750_ONE_H_RES   0x20   /* 单次高分辨率模式 */
#define BH1750_ONE_H_RES2  0x21   /* 单次高分辨率模式2 */
#define BH1750_ONE_L_RES   0x23   /* 单次低分辨率模式 */

/**
 * 向 BH1750 写入命令
 * @param fd   文件描述符
 * @param cmd  命令字节
 * @return     成功返回0，失败返回-1
 */
static int bh1750_write_cmd(int fd, unsigned char cmd)
{
    if (write(fd, &cmd, 1) != 1) {
        perror("写入 BH1750 命令失败");
        return -1;
    }
    return 0;
}

/**
 * 从 BH1750 读取光照强度（单位：lux）
 * @param fd   文件描述符
 * @return     光照强度值（lux），失败返回 -1.0
 */
static float bh1750_read_lux(int fd)
{
    unsigned char buf[2] = {0};

    if (read(fd, buf, 2) != 2) {
        perror("读取 BH1750 数据失败");
        return -1.0;
    }

    /* 原始值 = buf[0] 高8位 | buf[1] 低8位，除以 1.2 得到 lux 值 */
    float lux = (float)((buf[0] << 8) | buf[1]) / 1.2;
    return lux;
}

int main(void)
{
    int fd;
    float lux;

    /* 1. 打开 I2C 设备 */
    fd = open(I2C_DEV, O_RDWR);
    if (fd < 0) {
        perror("打开 I2C 设备失败");
        return -1;
    }

    /* 2. 设置从设备地址 */
    if (ioctl(fd, I2C_SLAVE, BH1750_ADDR) < 0) {
        perror("设置 I2C 从设备地址失败");
        close(fd);
        return -1;
    }

    /* 3. 发送上电命令 */
    if (bh1750_write_cmd(fd, BH1750_POWER_ON) < 0) {
        close(fd);
        return -1;
    }

    /* 4. 设置为连续高分辨率模式 */
    if (bh1750_write_cmd(fd, BH1750_CONT_H_RES) < 0) {
        close(fd);
        return -1;
    }

    /* 5. 循环读取光照强度 */
    printf("BH1750 光照强度传感器已启动，按 Ctrl+C 退出\n");
    while (1) {
        usleep(180000);  /* 等待 180ms（测量时间 120ms + 余量） */

        lux = bh1750_read_lux(fd);
        if (lux >= 0) {
            printf("光照强度: %6.2f lux\n", lux);
            fflush(stdout);
        }
    }

    close(fd);
    return 0;
}
```

### 4.2 方案二：Python 实现（简单快捷）

适合快速原型开发和测试。

#### 4.2.1 安装依赖

```bash
sudo apt-get install -y python3-smbus
```

#### 4.2.2 完整代码 `bh1750_read.py`

```python
import smbus
import time

# I2C 总线号（树莓派使用 1）
I2C_BUS = 1
# BH1750 I2C 地址（ADDR 接 GND）
BH1750_ADDR = 0x23

# BH1750 指令
POWER_ON      = 0x01
CONT_H_RES   = 0x10  # 连续高分辨率模式, 1lx, 120ms
CONT_H_RES2  = 0x11  # 连续高分辨率模式2, 0.5lx, 120ms
CONT_L_RES   = 0x13  # 连续低分辨率模式, 4lx, 16ms


def bh1750_init(bus):
    """初始化 BH1750：上电 + 设置测量模式"""
    bus.write_byte(BH1750_ADDR, POWER_ON)
    bus.write_byte(BH1750_ADDR, CONT_H_RES)


def bh1750_read_lux(bus):
    """读取光照强度，返回 lux 值"""
    data = bus.read_i2c_block_data(BH1750_ADDR, CONT_H_RES, 2)
    raw = (data[0] << 8) | data[1]
    lux = raw / 1.2
    return lux


def main():
    bus = smbus.SMBus(I2C_BUS)
    bh1750_init(bus)

    print("BH1750 光照强度传感器已启动，按 Ctrl+C 退出")
    try:
        while True:
            time.sleep(0.18)  # 等待测量完成
            lux = bh1750_read_lux(bus)
            print(f"光照强度: {lux:6.2f} lux")
    except KeyboardInterrupt:
        print("\n已停止采集")
    finally:
        bus.close()


if __name__ == "__main__":
    main()
```

#### 4.2.3 运行

```bash
sudo python3 bh1750_read.py
```

### 4.3 代码接口说明

以下为 BH1750 核心接口函数的调用流程与说明：

#### 4.3.1 初始化流程

```
打开 I2C 设备 (/dev/i2c-1)
       │
       ▼
设置从设备地址 (ioctl: I2C_SLAVE, 0x23)
       │
       ▼
发送上电命令 (写入 0x01)
       │
       ▼
设置测量模式 (写入 0x10: 连续高分辨率)
       │
       ▼
等待 180ms（首次测量完成）
       │
       ▼
循环读取 2 字节数据 → 计算 lux
```

#### 4.3.2 BH1750 常用指令表

| 指令码 | 含义 | 测量时间 | 分辨率 |
|--------|------|----------|--------|
| 0x00 | 断电模式 | - | - |
| 0x01 | 上电模式 | - | - |
| 0x07 | 重置数据寄存器 | - | - |
| 0x10 | 连续高分辨率模式 | 120ms | 1 lx |
| 0x11 | 连续高分辨率模式2 | 120ms | 0.5 lx |
| 0x13 | 连续低分辨率模式 | 16ms | 4 lx |
| 0x20 | 单次高分辨率模式 | 120ms | 1 lx |
| 0x21 | 单次高分辨率模式2 | 120ms | 0.5 lx |
| 0x23 | 单次低分辨率模式 | 16ms | 4 lx |

#### 4.3.3 光照值计算公式

```
光照强度 (lux) = (高字节 << 8 | 低字节) / 1.2
```

其中 BH1750 返回 2 字节无符号整数，`buf[0]` 为高字节，`buf[1]` 为低字节。

## 5. 原始源码说明

原始源码是为 UP-Tech i.MX8 开发板编写的，包含两部分：

### 5.1 内核驱动 (`driver/BH1750_driver.c`)

- 作用：将 DVI 引脚设为高电平以使能传感器
- 使用 platform driver 框架，匹配设备树节点 `fsl,p8-ports`
- **树莓派上不需要此驱动**，直接将 DVI 引脚接 3.3V 即可

### 5.2 测试程序 (`test/BH1750_test.c`)

- 通过 `/dev/i2c-1` 直接读写 I2C 总线
- 设置 7-bit 地址模式，从设备地址 `0x23`
- 发送上电指令 `0x01`，设置连续高分辨率模式 `0x10`
- 每 120ms 读取一次 2 字节数据，计算并打印 lux 值
- **此代码的核心逻辑可以直接在树莓派上复用**（本文档方案一即基于此改写）

## 6. 常见问题排查

| 问题 | 排查方法 |
|------|----------|
| `i2cdetect` 看不到 0x23 设备 | 检查接线是否正确，SCL/SDA 是否接反；确认 ADDR 引脚接了 GND；检查 DVI 引脚是否接了 3.3V |
| 打开 `/dev/i2c-1` 报权限错误 | 使用 `sudo` 运行程序，或将用户加入 i2c 组：`sudo usermod -aG i2c $USER` |
| 读取数据始终为 0 | 确认发送了上电命令 0x01 和模式设置命令 0x10；检查等待时间是否足够（>120ms） |
| 数据波动大 | 正常现象，可对连续多次读数取平均值平滑 |
| 编译报找不到 `linux/i2c-dev.h` | 安装头文件：`sudo apt-get install libi2c-dev` |
